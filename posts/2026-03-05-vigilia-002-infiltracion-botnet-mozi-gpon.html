REPORTE DE ANÁLISIS FORENSE Y SEGURIDAD

El equipo de Vigilia Abyssi ha documentado un intento de intrusión activo dirigido a infraestructura crítica corporativa. El ataque explota una vulnerabilidad conocida en el framework Apache Struts, permitiendo la ejecución remota de código (RCE). El objetivo del atacante fue la descarga y despliegue de binarios asociados al botnet Mozi, diseñado para minería criptográfica no autorizada y participación en ataques DDoS.

La vulnerabilidad identificada corresponde a CVE-2018-10561. Esta falla reside en la forma en que Apache Struts 2 procesa las solicitudes HTTP, específicamente en el manejo de encabezados OGNL (Object-Graph Navigation Language). Un atacante puede inyectar expresiones maliciosas a través del encabezado Content-Type o User-Agent, logrando ejecutar comandos arbitrarios en el sistema operativo subyacente del servidor web.

La explotación exitosa de esta vulnerabilidad otorga al adversario privilegios equivalentes al usuario que ejecuta el servicio de aplicación (generalmente un usuario con permisos elevados como 'tomcat' o 'www-data').

Se ha interceptado tráfico saliente y entrante asociado a la dirección IP 102.212.40.115. El análisis de los logs del servidor web revela una solicitud HTTP malformada diseñada para evadir filtros básicos de WAF (Web Application Firewall).

El comando ejecutado tras la explotación fue:

wget Mozi.m

Este comando indica la descarga directa de un binario desde un servidor C2 (Command and Control) externo. El archivo 'Mozi.m' es característico de las campañas masivas observadas en el sector industrial y gubernamental durante los últimos ciclos de amenazas. Una vez descargado, este binario se ejecuta silenciosamente para establecer persistencia y conectar la víctima a la red de bots.

La materialización de este ataque conlleva las siguientes consecuencias:

- Compromiso total del servidor afectado.

- Consumo excesivo de recursos computacionales debido a procesos de minería en segundo plano.

- Uso de la infraestructura comprometida para lanzar ataques distribuidos contra terceros objetivos.

- Riesgo de exfiltración de datos sensibles almacenados localmente o accesibles desde el servidor.

Se han implementado las siguientes acciones inmediatas:

- Aislamiento del segmento de red afectado para prevenir la propagación lateral.

- Actualización inmediata de Apache Struts a una versión parcheada (2.5.30 o superior).

- Bloqueo de la dirección IP 102.212.40.115 en el firewall perimetral y en los sistemas de detección de intrusos (IDS/IPS).

- Eliminación del archivo descargado 'Mozi.m' y revisión de cron jobs o servicios iniciados recientemente para asegurar la eliminación de persistencia.

Se recomienda a la dirección técnica realizar un escaneo exhaustivo de todos los activos que ejecuten versiones vulnerables de Apache Struts. Además, se debe reforzar la segmentación de red y habilitar el registro detallado (logging) para facilitar la detección temprana de anomalías en el futuro. La vigilancia continua sobre indicadores de compromiso (IOCs) relacionados con el botnet Mozi es imperativa.
